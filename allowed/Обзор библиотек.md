Обзор библиотек

# Назначение

Этот документ описывает библиотеки, разрешённые для проектов Human Horizon.
Он служит кратким обзором и навигацией. Детальные правила по категориям находятся в папке libraries.

## Структура

Обзор библиотек — общий список разрешённых библиотек и краткие правила применения.
libraries — папка с детальными спецификациями по категориям.
Новые категории библиотек добавляются в libraries отдельными документами.

## Этот документ не должен дублировать детальные правила из libraries.

Web и WebView UI
Solid
Статус: разрешено.
Применение:
• UI для web
• UI для webview
• переносимый интерфейсный слой на HTML, CSS и JavaScript
Ограничения:
• не использовать как нативную оболочку приложения

## • не считать заменой мобильной или десктопной среды выполнения

Tailwind CSS
Статус: разрешено.
Применение:
• стилизация web и webview UI
• utility-first CSS-подход
• работа вместе с design tokens
Ограничения:
• не использовать без design tokens
• держать кастомные стили минимальными

## Tailwind utilities

tailwind-merge разрешён для безопасного объединения Tailwind-классов.
tailwind-variants разрешён для типобезопасных variants и compound variants.

## Terminal UI

В этом обзоре не дублируются правила для bubbletea, charmbracelet/x/ansi и creack/pty.
TUI-библиотеки можно использовать только если проект явно объявляет terminal таргет.
Правило добавления библиотек
Новая библиотека не считается разрешённой только потому, что похожа на уже разрешённую.
Перед добавлением новой библиотеки нужно обновить этот обзор или соответствующий документ в libraries.
Если библиотека вводит новую среду выполнения, архитектурное предположение или новый таргет, сначала нужно добавить решение в decisions.
