# Инструменты

Разрешённые инструменты разработки.

## pnpm

Статус: **предпочтительно** (разрешено, рекомендуется)  
Область применения:

- управление зависимостями в проектах    
- замена npm и yarn для всех новых проектов    
- workspace-управление для монорепозиториев

Совместимость:

- Node.js    
- TypeScript-проекты    
- frontend/backend

Ограничения:

- **не использовать npm** — pnpm является предпочтительным менеджером пакетов    
- **не использовать yarn** — только pnpm для новых проектов

Обоснование:

- жёсткое связывание зависимостей (peer dependencies работают корректно)    
- эффективное использование диска (shared node_modules)    
- точность lock-файла (pnpm-lock.yaml)    
- встроенная поддержка workspace

## Vitest

Статус: разрешено  
Область применения:

- unit tests    
- integration tests    
- test tooling для TypeScript

Совместимость:

- TypeScript-проекты    
- Node.js    
- frontend tooling

Ограничения:

- использовать как основной test runner в TypeScript-частях проекта    
- не вводить параллельный тестовый стек без отдельного добавления в allowlist

Обоснование:

- подходит для современного TypeScript toolchain    
- хорошо сочетается с shared logic и frontend/backend кодом

## StrykerJS

Статус: разрешено  
Область применения:

- mutation testing    
- quality verification для TypeScript и JavaScript

Совместимость:

- TypeScript    
- JavaScript    
- Vitest    
- Node.js

Ограничения:

- использовать как слой mutation testing поверх существующих тестов    
- не заменяет unit и integration tests

Обоснование:

- подходит для проверки качества тестов    
- даёт mutation testing для TypeScript-стека

## Go toolchain

Статус: разрешено  
Область применения:

- сборка Go-проектов    
- тестирование    
- форматирование    
- линтинг

### go test

Встроенный test runner. Table-driven тесты — стандартный паттерн.  
go test ./...          # все тесты  
go test -v ./...       # verbose  
go test -run TestName  # конкретный тест  
go test -bench .       # бенчмарки

### go fmt / gofumpt

Форматирование кода.  
go fmt ./...           # стандартный форматтер  
go install mvdan.cc/gofumpt@latest && gofumpt -l -w .  # более строгий

### golangci-lint

Статус: разрешено  
Область применения:

- линтинг Go-кода    
- проверка стиля, ошибок, безопасности

go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest  
golangci-lint run ./...

### go vet

Встроенный статический анализатор.  
go vet ./...  
Ограничения:

- не вводить альтернативные Go-тулы без отдельного добавления в allowlist

Обоснование:

- встроенный toolchain — минимальные зависимости    
- go test — быстрые тесты без сторонних фреймворков    
- golangci-lint — стандартный линтер в Go-сообществе
