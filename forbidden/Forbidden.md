Запрещено

# Назначение

Этот документ фиксирует явные запреты Human Horizon.

Главное правило простое: всё запрещено, если явно не разрешено в спецификациях.

Отсутствие запрета не означает разрешение. Разрешение появляется только через allowed, architecture, targets, guides или принятое решение в decisions.

# Что запрещено по умолчанию

• новые языки без разрешения
• новые фреймворки без разрешения
• новые библиотеки без разрешения
• новые инструменты без разрешения
• новые архитектурные паттерны без решения
• новые предположения о среде выполнения без решения
• новые сетевые протоколы без решения
• новые механизмы хранения без решения
• новые системы сборки без решения

# TypeScript

Не использовать any без явно описанного исключения.
Не отключать проверку типов.
Не подавлять ошибки типов вместо исправления модели.
Не добавлять второй тестовый стек без обновления разрешительного списка.

# Архитектура

Не добавлять скрытое глобальное изменяемое состояние.
Не добавлять неявный dependency injection.
Не использовать monkey patching.
Не вводить недокументированные соглашения.
Не дублировать источник истины.
Не допускать утечки платформенно-специфичного кода в общие слои.

# Процесс

Не обходить TDD для production-кода.
Не добавлять абстракции без текущего сценария использования.
Не добавлять зависимости до обновления allowed или decisions.
Не менять публичное поведение без обновления документации.

# Менеджеры пакетов

Для новых проектов используется pnpm.
npm и yarn не разрешены для новых проектов, если отдельное решение явно не говорит обратного.
