## Управление состоянием

### Принцип: Единый источник истины

Всё состояние должно иметь единственный источник истины. Не синхронизируй состояние между разными хранилищами.

```typescript
// ❌ ЗАПРЕЩЕНО: Дублирование состояния
const [user, setUser] = useState<User>(initialUser);
const [userName, setUserName] = useState(initialUser.name);

// ✅ Единый источник
const [user, setUser] = useState<User>(initialUser);
const userName = user.name; // Вычисляемое значение, не хранимое
```

### State Machine для сложных состояний

Для сложных состояний используй state machine:

```typescript
type OrderState =
  | { status: 'draft' }
  | { status: 'pending_payment'; order: Order }
  | { status: 'paid'; order: Order }
  | { status: 'processing'; order: Order }
  | { status: 'shipped'; order: Order; trackingNumber: string }
  | { status: 'delivered'; order: Order }
  | { status: 'cancelled'; order: Order; reason: string };

function nextState(current: OrderState, event: Event): OrderState {
  // Каждый переход явный и валидный
  switch (current.status) {
    case 'draft':
      if (event.type === 'submit')
        return { status: 'pending_payment', order: current.order }
      break
    // ...
  }
}
```

## Дизайн функций

### Функции должны быть тотальными

Функция должна обрабатывать все возможные входные значения. Не используй исключения.

```typescript
// ❌ НЕ тотальная функция
function divide(a: number, b: number): number {
  if (b === 0) throw new Error('Division by zero') // Исключение!
  return a / b
}

// ✅ Тотальная функция
function divide(a: number, b: number): Result<number, 'division_by_zero'> {
  if (b === 0) return { ok: false, error: 'division_by_zero' }
  return { ok: true, value: a / b }
}
```

### Композиция функций

Строй сложную логику из простых функций:

```typescript
const pipe = <T>(...fns: ((arg: T) => T)[]) => (value: T): T =>
  fns.reduce((acc, fn) => fn(acc), value)

const processUser = pipe(
  validateUser,
  normalizeEmail,
  enrichWithDefaults,
  serializeForStorage
)
```

## Запрещённые паттерны

### 1. any

```typescript
// ❌ ЗАПРЕЩЕНО
const data: any = getData()

// ✅
const data = getData() as unknown as User
```

### 2. Non-null assertion

```typescript
// ❌ ЗАПРЕЩЕНО
const name = user!.name

// ✅
const name = user.name // Если тип Optional, обрабатывай оба случая
```

### 3. Типы с optional полями

```typescript
// ❌ ЗАПРЕЩЕНО
interface User {
  name?: string
  email?: string
}

// ✅
type User = { name: string; email: string } | { name?: never; email?: never }
```

### 4. catch с any

```typescript
// ❌ ЗАПРЕЩЕНО
try {
  doSomething()
} catch (e) {
  console.error(e)
}

// ✅
try {
  doSomething()
} catch (e) {
  return { ok: false, error: e instanceof Error ? e : new Error(String(e)) }
}
```

## Дополнительные паттерны

### Template Literal Types

```typescript
type HTTPMethod = 'GET' | 'POST' | 'PUT' | 'DELETE'
type Endpoint = '/users' | '/posts' | '/comments'
type Route = `${HTTPMethod} ${Endpoint}`

function request<T>(route: Route): Result<T, Error> {
  // Тип route гарантирует корректный формат
}
```

### Const Assertions

```typescript
const routes = {
  home: '/',
  about: '/about',
  contact: '/contact'
} as const

type Route = typeof routes[keyof typeof routes] // '/' | '/about' | '/contact'
```

### Assert Functions

```typescript
function assertIsDefined<T>(value: T): asserts value is NonNullable<T> {
  if (value === null || value === undefined) {
    throw new Error('Value is not defined')
  }
}
```
