# Методология Однозначной Разработки

> **Философия**: Сделай невалидные состояния невыразимыми.

## Основные Принципы

### 1. Парси, Не Валидируй

```typescript
// ❌ Валидировать и надеяться
function processUser(data: unknown) {
  if (typeof data === 'object' && 'email' in data) {
    console.log(data.email)
  }
}

// ✅ Парсить в валидированный тип
function parseUser(data: unknown): Result<ValidUser, ValidationError> { }
function processUser(user: ValidUser) {
  console.log(user.email) // Безопасно!
}
```

### 2. Невалидные Состояния Невыразимы

```typescript
// ❌ Невалидные состояния возможны
interface ApiResponse {
  status: 'loading' | 'success' | 'error'
  data?: User
  error?: Error
}

// ✅ Невалидные состояния невозможны
type ApiResponse =
  | { status: 'loading' }
  | { status: 'success'; data: User }
  | { status: 'error'; error: Error }
```

### 3. Ошибки как Значения

type Result<T, E> =
  | { ok: true; value: T }
  | { ok: false; error: E }

### 4. Брендированные Типы

```typescript
type Email = string & { readonly __brand: 'Email' }

function createEmail(value: string): Result<Email, ValidationError> {
  if (!value.match(/^[^\s@]+@[^\s@]+\.[^\s@]+$/)) {
    return { ok: false, error: new ValidationError('Invalid email') }
  }
  return { ok: true, value: value as Email }
}
```

### 5. Проверка Полноты

```typescript
type Status = 'pending' | 'approved' | 'rejected'

function handleStatus(status: Status): string {
  switch (status) {
    case 'pending': return 'Waiting'
    case 'approved': return 'Approved'
    case 'rejected': return 'Rejected'
    default:
      const _exhaustive: never = status
      return _exhaustive
  }
}
```

## Вспомогательные Типы

```typescript
type Result<T, E> = { ok: true; value: T } | { ok: false; error: E }
type Option<T> = { some: T } | { none: true }
type NonEmptyArray<T> = [T, ...T[]]
type Brand<T, B> = T & { readonly __brand: B }
```

## Золотые Правила

1. Невалидные состояния невыразимы
1. Парси, не валидируй
1. Ошибки как значения (Result)
1. Брендируй примитивы
1. Проверка полноты везде
1. Никакого any
1. Иммутабельность по умолчанию
