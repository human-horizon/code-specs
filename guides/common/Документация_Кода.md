## TypeScript: JSDoc + TypeDoc

### JSDoc для публичного API

```typescript
/**
 * Создаёт нового пользователя с проверенными данными.
 *
 * @param params - Параметры создания пользователя
 * @returns Result с созданным пользователем или ошибкой валидации
 *
 * @example
 * ```typescript
 * const result = createUser({
 *     email: 'user@example.com',
 *     name: 'Alice',
 * })
 * ```
 */
function createUser(params: CreateUserParams): Result<User, ValidationError> {
    // implementation
}
```

### Документация типов

```typescript
/**
 * Представляет проверенного пользователя в системе.
 */
interface User {
    /** Уникальный идентификатор (UUID v4) */
    id: UserId
    /** Проверенный email-адрес */
    email: Email
    /** Отображаемое имя */
    name: string
}
```

### Обязательные теги

| Тег | Когда |
|-----|-------|
| @param | Все параметры |
| @returns | Все функции с возвратом |
| @example | Публичные функции |
| @pure | Функция без побочных эффектов |
| @sideeffect | Каждый побочный эффект (db, fs, net, log, mutation) |
| @throws | Если может выбросить ошибку |

### Side Effects Contract

Каждая функция **должна** декларировать свои побочные эффекты через JSDoc. Это обязательно для AST-анализа кода.

```typescript
/**
 * Сохраняет пользователя в базу данных.
 * @sideeffect db — INSERT в таблицу users
 * @sideeffect log — запись в audit log
 * @throws {ValidationError} если email невалиден
 */
async function saveUser(user: User): Promise<void> { ... }

/**
 * Вычисляет общую стоимость.
 * @pure
 */
function calculateTotal(items: Item[]): number { ... }
```

**Правила:
**

- @pure — нет побочных эффектов, нет I/O, нет мутаций
- @sideeffect <type> — декларирует внешний эффект: db, fs, net, log, mutation
- @throws {Type} — декларирует выбрасываемые исключения
- Если нет ни @pure, ни @sideeffect — функция **недокументирована** и считается нарушением

// ✅ Объясняет ПОЧЕМУ
